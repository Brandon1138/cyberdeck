logos, or other project branding to imply endorsement or official status.

You may use the name truthfully to describe compatibility, origin, or an
unmodified copy of the project. Modified distributions and independent services
must use distinct branding and must not suggest that they are maintained,
certified, or endorsed by the Cyberdeck project.

Examples of permitted descriptive use include "based on Cyberdeck" and
"compatible with Cyberdeck." Examples requiring separate permission include
using Cyberdeck as the primary name of a modified distribution or commercial
service, or using project artwork as that offering's identity.

Questions about branding may be opened as a private maintainer contact through
the repository's GitHub Security Advisory interface.
